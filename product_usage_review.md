# Product Usage Review

Use this log while actively using the product to capture real UX, workflow, and reliability flaws.
Each dated section should list actionable TODOs that agents can pick up and fix.

## How to use this file

- Add a new section per day in `MM/DD/YYYY` format (example: `04/06/2026`).
- Log issues as TODO items with clear scope and expected outcome.
- Keep each item specific enough to implement without extra clarification.
- Optionally tag each item with area labels such as `web`, `api`, `handler`, `store`, or `docs`.

## Suggested TODO format

- [ ] **Title:** Short issue summary
  - **Observed:** What happened while using the product
  - **Expected:** What should have happened
  - **Impact:** Why this matters (speed, trust, usability, bugs, etc.)
  - **Area:** `web` | `api` | `handler` | `store` | `docs`
  - **Notes:** Optional reproduction details
